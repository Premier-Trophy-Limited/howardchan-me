04Writing

Field notes from building solo with AI agents — real findings, with the numbers attached.

a commit is a checkpoint

2026-06-25

A commit is a checkpoint, and shipping only starts after live verification exposes the hidden failures.

when is a change actually done?

2026-06-24

A change is done when the live service is verified end to end, because repo-local green can create false confidence.

your offline backup is not your source of truth

2026-06-23

A backup you write to is just a second thing to keep in sync; a backup you only read from in a disaster actually saves you.

a green checkmark in your repo is not proof of anything

2026-06-22

A green checkmark in the repo can hide a stale live endpoint, and the only reliable receipt is a live request you make yourself.

shipped a feature last week

2026-06-20

a green build and clean merge can still miss the user path; if nobody walked it live, the feature was not shipped yet.

production verification is part of done

2026-06-19

production verification belongs in the definition of done, because the live check is where the answer shows up.

one source of truth for repos, services, creds, automations

2026-06-18

A single source of truth keeps repos, services, access details, and automations in sync by updating the record with the change.

Moving every secret into one canonical store made our env files boring again

2026-06-17

Moving secrets into one canonical store made env files boring again and made rotation and drift easier to manage.
